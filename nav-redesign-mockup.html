Project / Sprint navigation — 3 design options

Tham khảo ClickUp left sidebar pattern. Cùng một stack thông tin (2 projects · mỗi project vài sprints · current sprint highlight), 3 cách bố trí khác nhau về screen real estate vs discoverability. Sample data: project My Project đang active sprint Sprint 42; project khác là godot framework.

A

Sidebar Native — bám sát ClickUp

Left rail 232px, Projects expand → Sprints lồng vào. Highlight rust-accent + inset border-left cho sprint đang chọn. Mọi thứ trên 1 view, không cần dropdown.

Ưu

  • Mọi project + sprint thấy một lần, không cần click mới biết có gì
  • Quen thuộc nhất (ClickUp/Linear/Notion users hiểu ngay)
  • Drag-drop reorder sprint dễ thêm sau

Nhược

  • Chiếm 232px chiều ngang vĩnh viễn — task table hẹp đi
  • Single-user app với 2-3 projects thì hơi over-engineered

Phù hợp khi

  • Có ≥3 projects và mỗi project ≥4 sprints
  • User muốn jump cross-project nhanh
  • Future: nested lists/folders trong project
B

Top Bar + Sprint Pills — Linear/Vercel-style

Compact project switcher trên góc trái (click → menu chọn project), sprint thành hàng pills bên dưới. Không sidebar — task area giữ nguyên width tối đa.

  • Sprint switch 1 click, không phải mở dropdown
  • Hợp với calm utility — minh bạch, không ồn

Nhược

  • Pills hàng dưới scroll horizontal khi >6 sprints — hơi phiền
  • Project switcher vẫn là dropdown (cùng cost như hiện tại)

Phù hợp khi

  • 1-3 projects, mỗi project 4-8 sprints active gần đây
  • User chủ yếu làm 1 project, đổi sprint nhiều hơn đổi project
  • Muốn giữ DNA "speed > breadth" của plan-tmp
C

Icon Rail + Panel — Slack/Discord hybrid

Slim 60px rail với project initials (rust badge cho current). Click rail → 224px secondary panel show sprint list. Compact, scale tốt với nhiều projects.

Ưu

  • Project switch 1 click trên rail — fastest cross-project
  • Scale tốt: 10+ projects vẫn vừa rail
  • Cảm giác "workspace" có cấu trúc, hợp khi sau này thêm Inbox / My Tasks

Nhược

  • Tốn 284px (60 rail + 224 panel) — task area hẹp nhất 3 options
  • 2 cấp navigation hơi phức tạp cho single-user
  • Panel có thể collapse được nhưng tăng thêm state

Phù hợp khi

  • Tương lai có nhiều projects (5+), hoặc thêm Inbox/Reports global
  • User quen Slack/Discord/Linear desktop
  • Sẵn sàng tradeoff width để có structure

Recommendation: Với DNA hiện tại của plan-tmp (single-user · 1-3 projects · "speed > breadth") → Option B hợp nhất. Nó giữ task area rộng, sprint switch 1-click không cần dropdown, và đổi project vẫn là dropdown (cùng cost với header hiện tại — chỉ thay layout của sprint picker).

Nếu kế hoạch dài hạn là nhiều projects + nested lists/folders trong project → chuyển luôn sang Option A, đỡ phải refactor 2 lần.

Tao có thể: 1) implement Option B vào app/src/App.tsx luôn, 2) mix attributes (vd: rail của C + pills của B), hoặc 3) show thêm variation thứ 4 (combo / dark mode / mobile crop). Tell tao đi nào.
